feat: SOL settlement option (USDC | SOL toggle)

The differentiator deepens. Now every settlement supports two assets:
- USDC (default, stable, recommended)
- SOL (native, no token account needed, with live USD conversion)

Schema v5
- settlements.asset (default 'USDC')
- settlements.asset_amount_base_units (text, holds lamports for SOL or USDC base units)
- index on asset

New libs
- lib/solana/sol.ts: SystemProgram.transfer builders (single + bulk, atomic)
- lib/price.ts: free CoinGecko SOL/USD feed with 30s in-memory cache
- centsToLamports + formatSol helpers

UI
- SettleUpButton: USDC|SOL pill toggle, lazy SOL price load, live conversion preview ("$50 = 0.3205 SOL"), context-aware CTA label
- SettleAllButton (bundle): same toggle. SOL bundle uses buildBulkSolTransferTx (N SystemProgram.transfer ix in one tx)
- Activity feed: asset badge per settlement (USDC green vs SOL violet), shows on-chain amount with USD equivalent below

Behavior
- recordSettlement persists asset + asset_amount_base_units
- USDC remains the default and recommended asset
- SOL fallback to USDC if price feed fails
- Settle-all atomicity preserved for both assets
